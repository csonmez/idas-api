# Request Validation Test Matrix

Bu matris Request Validation fazinda eklenecek testleri tarif eder. Mevcut test altyapisi `node:test`, `node:assert/strict`, `supertest` ve `tsx --test src/**/*.test.ts` uzerindedir. En dusuk riskli pattern, production router'a eklenmeyen kucuk `express()` test app'leri ve gercek `errorHandler` ile integration test yazmaktir.

Async refinement destegi kaynak planda zorunlu acceptance criterion degildir. Bu nedenle async schema davranisi zorunlu test degil, acik karar olarak degerlendirilmelidir.

## Test Notları

- `src/app.test.ts` mevcut Error Contract ve app lifecycle testlerini tasir. Request validation testleri yeni dosyada tutulursa dosya daha odakli kalir.
- Test-only route'lar production `src/routes/index.ts` icine eklenmemelidir.
- Handler calismama beklentisi bir sentinel degisken veya counter ile kanitlanabilir.
- Request mutation beklentisi icin success route response'u parsed request alanlarini ve untouched alan sentinel'lerini dondurebilir.
- `req.query` mutation Express 5.2.1 getter kısıtı nedeniyle ozellikle integration testle dogrulanmalidir.
- Nested ve array field path formatlari kaynaklarda kesinlesmedigi icin testler implementasyon oncesi secilen formatla uyumlu hale getirilmelidir.
